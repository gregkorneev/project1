# ДЗ-1, Вариант 14
## Машинное обучение и алгоритмы кодирования информации

Данный репозиторий содержит реализацию двух заданий домашней работы и организован в виде монорепозитория с единой системой сборки и общим меню запуска.

## Состав работы

1. Задание 1 (ML) — выявление скрытых факторов успеха студентов
   Алгоритм: факторный анализ

2. Задание 2 — алгоритм кодирования информации
   Алгоритм: Triple DES (3DES)

---

## Задание 1 — Факторный анализ (ML)

В первой части реализован факторный анализ данных студентов с целью выявления скрытых факторов, влияющих на их успеваемость.

Основные этапы алгоритма:
- чтение CSV-датасета;
- стандартизация признаков;
- вычисление корреляционной матрицы;
- вычисление собственных значений и собственных векторов;
- формирование факторных нагрузок;
- сохранение результатов в CSV-файлы.

После выполнения C++-части автоматически запускаются Python-скрипты визуализации:
- тепловая карта корреляционной матрицы;
- график собственных значений;
- визуализация факторных нагрузок.

Результаты сохраняются в каталоге ML-part1/results.

---

## Задание 2 — Triple DES (3DES)

Во второй части реализован алгоритм симметричного блочного шифрования Triple DES в режиме CBC с выравниванием PKCS#7.

Возможности программы:
- шифрование файла с данными студентов;
- дешифрование зашифрованного файла;
- автоматическое создание каталога results;
- самопроверка корректности (decrypt(encrypt(data)) == data);
- замеры времени шифрования и дешифрования.

Используется фиксированный ключ шифрования, заданный в коде программы (учебная реализация).

---

## Сборка проекта и запуск проекта

```rm -rf build && cmake -B build && cmake --build build && ./build/bin/launcher ```

---

## Успешный запуск
На экране появится меню:

1 - Запустить задачу 1 (ML) + визуализации
2 - Запустить задачу 2 (3DES)
0 - Выход

При выборе пункта 1 выполняется C++-реализация факторного анализа, после чего автоматически запускаются Python-скрипты построения графиков.

При выборе пункта 2 запускается программа Triple DES с меню шифрования и дешифрования данных.

---

## Примечания

Каталог build не хранится в репозитории и игнорируется системой контроля версий.
Каталоги results создаются автоматически при выполнении программ.
Все относительные пути корректно работают за счёт запуска программ из соответствующих подкаталогов.
