# Задание 2 (ДЗ-1, Вариант 14)
## Алгоритм кодирования информации — Triple DES (3DES)

В данном проекте реализован алгоритм симметричного блочного шифрования **Triple DES (3DES)**.
Программа предназначена для демонстрации шифрования и дешифрования данных студентов в рамках учебного задания.

В качестве входных данных используется датасет `students.csv`, применявшийся в задании 1.

---

## Используемый алгоритм

**Triple DES (3DES)** — это расширение алгоритма DES, при котором операция шифрования выполняется три раза
(схема EDE — Encrypt–Decrypt–Encrypt).

Основные характеристики алгоритма:
- тип алгоритма: симметричный, блочный;
- размер блока: 64 бита;
- режим работы: CBC (Cipher Block Chaining);
- выравнивание данных: PKCS#7;
- вычислительная сложность: O(n), где n — размер входных данных в байтах.

В реализации используется криптографическая библиотека **OpenSSL**.

---

## Описание файлов

- `data/students.csv` — входной датасет (данные студентов);
- `include/crypto3des.hpp` — объявления функций шифрования, дешифрования и работы с файлами;
- `src/crypto3des.cpp` — реализация алгоритма 3DES (CBC) с использованием OpenSSL;
- `src/main.cpp` — основной файл программы с текстовым меню;
- `results/` — каталог для сохранения зашифрованных и расшифрованных файлов.

---

## Сборка проекта

Перед сборкой необходимо, чтобы в системе была установлена библиотека OpenSSL (пакет `libssl-dev`).

Команда для сборки и запуска проекта:

```rm -rf build && rm -rf results && mkdir -p results && cmake -B build && cmake --build build  && ./build/three_des```
---
После запуска пользователю предлагается меню:

1 - Зашифровать файл
2 - Расшифровать файл
0 - Выход

---

## Принцип работы программы

При выборе пункта **1** программа:
- автоматически считывает файл `data/students.csv`;
- шифрует его алгоритмом 3DES;
- сохраняет результат в файл `results/students.enc`.

При выборе пункта **2** программа:
- считывает файл `results/students.enc`;
- выполняет дешифрование;
- сохраняет восстановленный файл в `results/students_dec.csv`.

В учебной реализации используется **фиксированный ключ шифрования**, заданный непосредственно в коде программы.
Это упрощает интерфейс и позволяет сосредоточиться на демонстрации принципов работы алгоритма.

---

## Проверка корректности

После шифрования автоматически выполняется самопроверка:

decrypt(encrypt(data)) == data

Успешное выполнение проверки подтверждает корректность реализации алгоритма.

---

## Просмотр зашифрованного файла

Зашифрованный файл `students.enc` имеет бинарный формат и не предназначен для просмотра как текст.

Для визуальной проверки корректности шифрования рекомендуется использовать просмотр файла
в шестнадцатеричном представлении.

Просмотр с помощью `hexdump`:

hexdump -C results/students.enc | head

Пример вывода:

00000000  70 63 53 bb 37 7c 85 c6  4c 0f e5 4e 09 f5 d8 aa
00000010  c2 6e 15 45 04 e0 e8 bd  03 f1 7b 59 34 ca da 1f

Такой вывод подтверждает, что файл:
- не содержит читаемых текстовых данных;
- полностью отличается от исходного CSV-файла;
- корректно зашифрован алгоритмом 3DES.

Дополнительная проверка:

strings results/students.enc | head
